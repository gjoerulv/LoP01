# Roadmap command guidance

When asked to plan the next work:
  1. Treat the repository baseline as whatever `CLAUDE.md` and `docs/implementation_roadmap.md` currently state.
  2. Latest completed milestone is recorded in `docs/implementation_roadmap.md`; do not hardcode an older milestone.
  3. Active scope cap is `docs/content_scope_v2.md`.
  4. Current selected milestone may be unset; use `docs/implementation_roadmap.md` §4/§5 as the source of truth.
  5. Read active docs before recommending anything, especially `docs/implementation_roadmap.md`, `docs/content_scope_v2.md`, and `CLAUDE.md`.
  6. Do not edit source/tests during a roadmap-only pass.
  7. Keep roadmap edits concise and avoid speculative milestone sprawl.
  8. Do not use archived `content_scope_v1.md` as the active scope after it is archived by the user.
