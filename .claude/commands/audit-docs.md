# Audit docs command guidance

Use this to check documentation after a milestone or planning pass.

Current baseline: **post-M24**.

Checklist:

- Active docs must not describe the repo as post-M23 or earlier.
- Latest completed milestone should be **M24 — v1 Strategic-Economy Proof Content** unless a later milestone is completed.
- Active scope cap should be `docs/content_scope_v2.md` after the user archives v1.
- M25 should be described as planned, not implemented, unless it has actually shipped.
- Archived docs are historical context only.
- Do not update broad vision/scope docs just to record milestone status; update them only when the playable content target or rules actually change.
- Prefer concise targeted edits over broad rewrites.
