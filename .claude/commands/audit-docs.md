Use this to check documentation after a milestone or planning pass.

Current baseline: read from `CLAUDE.md` and `docs/implementation_roadmap.md`.

Checklist:

- Active docs must not describe the repo as post-M26 or earlier unless explicitly referring to historical/archive context.
- Latest completed milestone should match `docs/implementation_roadmap.md`; as of the post-M27 baseline it is **M27 — Owned Service Overview / Strategic Service Readout**.
- Active scope cap should be `docs/content_scope_v2.md`.
- The current selected milestone may be unset; do not invent one if the roadmap has not selected it.
- Archived docs are historical context only.
- Do not update milestone-agnostic final-vision docs just to record milestone status; update them only when the long-term rule, schema, terminology, or architecture actually changes.
- Prefer concise targeted edits over broad rewrites.
